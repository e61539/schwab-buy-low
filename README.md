# Schwab Buy Low Launcher

Repo-local packaging for the staged buy-low process.

## What Is Included

- `buylow_new.py` - buy engine with staged triggers, ATR gates, caps, brakes, and Schwab order submission.
- `buy_relax_kit.py` - per-symbol sizing and override helpers.
- `trade_logger.py` - CSV trade logger with de-dupe state.
- `capital_readiness.py` - advisory-only Schwab/Merrill funding readiness JSON writer.
- `dashboard_api.py` - dependency-free local JSON API for dashboard/iPhone clients.
- `run_one_symbol_buy_quiet.ps1` - PowerShell wrapper for one symbol.
- `buysqg.cmd` - Windows launcher for SPY, QQQ, and GLD.
- `config/` - non-secret example strategy configuration.
- `runtime/` - local logs, locks, stage state, brakes, and generated files.

## Sensitive Files Not Committed

The `.gitignore` excludes broker tokens, account mappings, logs, locks, generated trade CSVs, and runtime state.

Create these locally when needed:

- `config/acct.json` from `config/acct.example.json`
- `config/merrill_reserve.json` from `config/merrill_reserve.example.json` if you want advisory Merrill reserve visibility
- `config/tokens.txt` generated by your Schwab auth flow
- environment variables `app_key` and `app_secret`

## Running

`buysqg.cmd` is repo-local and defaults to preview mode because `CONFIRM_FLAG` is empty.

```cmd
buysqg.cmd
```

For live orders, edit `buysqg.cmd` and set:

```cmd
set "CONFIRM_FLAG=--confirm"
```

Keep only one `--confirm` in the final argument list.

## Path Model

By default, all config and runtime files resolve relative to this repo folder:

- `config\buy.dic`
- `config\sym_caps.dic`
- `config\sym_overrides.json`
- `config\atrk.json`
- `config\tokens.txt` unless `BUYLOW_TOKENS_FILE` is set
- `runtime\logs` unless the launcher sets a shared log directory
- `runtime\locks`
- `runtime\equity_brake.json`
- `runtime\daily_alloc.json`
- `C:\temp\capital_readiness.json` advisory capital readiness output, when BuyLow sees a blocked/no-size opportunity

## Capital Readiness

Capital readiness is advisory only. It reads manually maintained Merrill reserve data and may write `C:\temp\capital_readiness.json` when Schwab cash, budget, or caps leave BuyLow unable to size an otherwise triggered opportunity.

Create a local Merrill reserve file when needed:

```powershell
copy .\config\merrill_reserve.example.json .\config\merrill_reserve.json
```

Edit market values manually. The real file is ignored by git. This feature does not call Merrill APIs, sell Merrill holdings, transfer funds, place trades, or raise BuyLow caps.

## Trend Rider Cooldowns

Trend Rider cooldowns are explainability and risk-control guardrails. They avoid duplicate buys, overtrading, repeated chasing of strength, and repeated signals during noisy consolidation while allowing trend confirmation after a signal or entry.

- Symbol cooldown: configured by `strategies\trend_rider\trend_config.json` key `cooldown_days`; blocks new-entry proposals for the same symbol.
- Add-on cooldown: configured by `add_on_cooldown_days`; blocks add-on proposals for the same symbol.
- Post-entry penalty: configured by `recent_purchase_penalty_days`; applies a score penalty after accepted signals or entries.
- Sector, portfolio, and post-exit cooldowns are not configured unless explicitly added later.

Override the readiness file path if the dashboard runs somewhere else, such as macOS:

```powershell
$env:BUYLOW_CAPITAL_READINESS_FILE = "C:\temp\capital_readiness.json"
```

On macOS/Linux:

```bash
export BUYLOW_CAPITAL_READINESS_FILE=/tmp/capital_readiness.json
```

Capital readiness endpoint:

```text
GET http://127.0.0.1:8765/api/capital-readiness
```

You can override the root with:

```powershell
$env:BUYLOW_HOME = "D:\path\to\schwab-buy-low"
```

The included `buysqg.cmd` points `BUYLOW_TOKENS_FILE` at `C:\temp\tokens.txt` and writes logs to `C:\temp\logs_ira1` so it can share token and log files with older launchers.
